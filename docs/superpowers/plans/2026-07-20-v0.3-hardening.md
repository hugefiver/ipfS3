# v0.3 Hardening Implementation Plan

> **For agentic workers:** Use the subagent-driven-development skill to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Complete all six v0.3 hardening items with AWS-compatible name validation, C2-safe SSE-C object and multipart authentication, CSPRNG chunk nonces, Head Range semantics, standard presigned GET/PUT evidence, CID response headers, and a deterministic real-TCP acceptance matrix.

**Architecture:** Keep the existing axum → s3s → `S3Impl` → ops → store/Kubo path and add checks at its current boundaries. Persist only a domain-separated HMAC fingerprint for SSE-C objects and multipart uploads, atomically claim a legacy object only after full ciphertext and size authentication, and extend the existing real-TCP/wiremock harness rather than introducing custom routes or alternate storage flows.

**Tech Stack:** Rust 2024 (MSRV 1.92), s3s 0.14.0, axum 0.8, sea-orm/sea-orm-migration 1.x, SQLite and PostgreSQL query builders, HMAC-SHA256, `subtle::ConstantTimeEq`, reqwest, wiremock, tokio, rust-s3.

**Global Constraints:**
- Start from `HEAD 276de042b29030195349fe91ac5ae8e944dcd591`; do not silently absorb later source changes.
- CID remains the S3 ETag for plain, SSE-S3, SSE-C, and completed multipart objects.
- Preserve the conservative pin policy: no multipart/delete unpin or CID reference-counting work.
- Never persist or log the raw SSE-C key, its base64 transport form, or its transport MD5.
- The configured master key must remain stable for the entire lifecycle of every SSE-C object and multipart upload; no key rotation protocol is in scope.
- Encrypted Range continues to decrypt the complete object and then slice plaintext; chunk-level Range is outside v0.3.
- Standard presigned requests continue through s3s query SigV4 and standard S3 operations; add no custom presigned route.
- Do not implement v0.4 remote pinning, public gateway configuration, or later roadmap work.
- Reuse dependencies already locked in `Cargo.lock`; install nothing and do not query Context7.
- Treat repository-local rustdoc as authoritative for s3s 0.14.0: `S3ServiceBuilder::set_validation(&mut self, impl NameValidation)`, `AwsNameValidation::new()`, `S3Response::with_headers(output, HeaderMap)`, and `HeadObjectInput::range: Option<Range>`.
- Do not edit `docs/superpowers/specs/2026-07-20-v0.3-hardening-design.md`.
- Each task uses RED → minimal implementation → GREEN. Do not stage or perform any Git write during a task.

```rust
pub struct TestHarness {
    pub endpoint: String,
    pub bucket: String,
    pub state: Arc<ipfs_s3_gateway::state::AppState>,
    pub kubo: wiremock::MockServer,
    pub observed_http: Arc<tokio::sync::Mutex<Vec<ObservedHttpRequest>>>,
    add_file_bytes: Arc<std::sync::Mutex<Vec<Vec<u8>>>>,
    cat_bodies: Arc<std::sync::RwLock<HashMap<String, Vec<u8>>>>,
}

impl TestHarness {
    pub fn captured_add_file_bytes(&self) -> Vec<Vec<u8>> {
        self.add_file_bytes.lock().expect("add capture mutex").clone()
    }

    pub fn set_cat_body(&self, cid: &str, body: Vec<u8>) {
        self.cat_bodies
            .write()
            .expect("cat body map")
            .insert(cid.to_owned(), body);
    }
}

fn kubo_add_file_bytes(request: &wiremock::Request) -> Vec<u8> {
    let content_type = request
        .headers
        .get(http::header::CONTENT_TYPE)
        .and_then(|value| value.to_str().ok())
        .expect("Kubo add Content-Type");
    let boundary = content_type
        .split("boundary=")
        .nth(1)
        .map(|value| value.trim_matches('"'))
        .expect("multipart boundary");
    let header_end = request
        .body
        .windows(4)
        .position(|window| window == b"\r\n\r\n")
        .expect("multipart file headers")
        + 4;
    let terminator = format!("\r\n--{boundary}").into_bytes();
    let file_end = request.body[header_end..]
        .windows(terminator.len())
        .position(|window| window == terminator.as_slice())
        .expect("multipart file terminator");
    request.body[header_end..header_end + file_end].to_vec()
}

fn kubo_cat_body(request: &wiremock::Request, body: Vec<u8>) -> Vec<u8> {
    let range = request
        .url
        .query_pairs()
        .find(|(name, _)| name == "bytes")
        .map(|(_, value)| value.into_owned());
    let Some(range) = range else {
        return body;
    };
    let (start, end) = range.split_once('-').expect("Kubo bytes=start-end");
    let start: usize = start.parse().expect("Kubo byte start");
    let end: usize = end.parse().expect("Kubo byte end");
    assert!(start <= end, "Kubo byte range is ascending");
    assert!(end < body.len(), "S3 range is checked before Kubo");
    body[start..=end].to_vec()
}
```

```rust
pub async fn complete_multipart_with_headers(
```

```rust
use hmac::{Hmac, Mac};
use subtle::ConstantTimeEq;

const SSE_C_FINGERPRINT_PREFIX: &str = "v1:hmac-sha256:";
const SSE_C_FINGERPRINT_DOMAIN: &[u8] = b"ipfs-s3-gateway/sse-c-key-fingerprint/v1";

impl MasterKey {
    pub fn sse_c_key_fingerprint(&self, key: &ObjectKey) -> String {
        let mut mac = Hmac::<Sha256>::new_from_slice(&self.bytes)
            .expect("HMAC-SHA256 accepts a 32-byte key");
        mac.update(SSE_C_FINGERPRINT_DOMAIN);
        mac.update(&[0]);
        mac.update(&key.bytes);
        format!(
            "{SSE_C_FINGERPRINT_PREFIX}{}",
            hex::encode(mac.finalize().into_bytes())
        )
    }

    pub fn verify_sse_c_key_fingerprint(
        &self,
        stored: &str,
        key: &ObjectKey,
    ) -> AppResult<bool> {
        let encoded = stored.strip_prefix(SSE_C_FINGERPRINT_PREFIX).ok_or_else(|| {
            AppError::Crypto("unsupported SSE-C key fingerprint format".to_owned())
        })?;
        let persisted = hex::decode(encoded)
            .map_err(|error| AppError::Crypto(format!("invalid SSE-C key fingerprint: {error}")))?;
        if persisted.len() != 32 {
            return Err(AppError::Crypto(format!(
                "SSE-C key fingerprint must be 32 bytes, got {}",
                persisted.len()
            )));
        }

        let candidate = self.sse_c_key_fingerprint(key);
        let candidate = hex::decode(
            candidate
                .strip_prefix(SSE_C_FINGERPRINT_PREFIX)
                .expect("generated fingerprint has the v1 prefix"),
        )
        .expect("generated fingerprint is lowercase hex");
        Ok(bool::from(persisted.ct_eq(candidate.as_slice())))
    }
}
```

Persisted corruption maps to `AppError::Crypto`/S3 `InternalError`; a well-formed fingerprint for a different key returns `false` so operations can map it to `InvalidArgument`.

```rust
async fn validate_sse_c_upload_key(
    state: &Arc<AppState>,
    upload_id: &str,
    headers: &http::HeaderMap,
) -> S3Result<crate::crypto::ObjectKey> {
    if determine_encryption_mode(headers)? != EncryptionMode::SseC {
        return Err(s3s::s3_error!(
            InvalidArgument,
            "a complete, unmixed SSE-C header set is required"
        ));

        state.store.db(),
        upload_id,
        &candidate,
    )
    .await?;

        s3s::s3_error!(InternalError, "SSE-C upload is missing its key fingerprint")
    })?;
    let matches = state
        .master_key
        .verify_sse_c_key_fingerprint(persisted, &object_key)
```

```rust
async fn authenticate_sse_c_parts(
    kubo: &crate::kubo::KuboClient,
    part_cids: &[String],
    key: Arc<crate::crypto::ObjectKey>,
) -> S3Result<()> {
    for cid in part_cids {
        let part_stream = crate::kubo::cat::stream_cat(kubo, cid, None)
            .await
            .map_err(|e| s3s::s3_error!(InternalError, "cat: {e}"))?;
        let decrypted = crate::crypto::chunker::decrypt_chunk_stream(part_stream, key.clone());
        tokio::pin!(decrypted);
        while let Some(chunk) = decrypted.next().await {
            match chunk {
                Ok(_) => {}
                Err(crate::error::AppError::Crypto(_)) => {
                    return Err(s3s::s3_error!(
                        InvalidPart,
                        "failed to decrypt part during complete — SSE-C key may not match the key used to upload parts"
                    ));
                }
                Err(error) => {
                    return Err(s3s::s3_error!(
                        InternalError,
                        "failed to stream encrypted multipart part: {error}"
                    ));
                }
            }
        }
    }
    Ok(())
}
```

```rust
pub fn encrypt_chunk_stream<S, E>(
    stream: S,
    key: Arc<ObjectKey>,
) -> impl Stream<Item = Result<Bytes, AppError>>
where
    S: Stream<Item = Result<Bytes, E>> + Unpin,
    E: Into<Box<dyn std::error::Error + Send + Sync>> + Send + 'static,
{

        let mut chunked = Box::pin(chunk_stream(stream));
        while let Some(chunk) = chunked.next().await {
            match chunk {
                Ok(plaintext) => {
                    let mut nonce = [0_u8; 12];

                    yield encrypt_chunk(&key, &nonce, &plaintext);
                }
                Err(error) => yield Err(AppError::Internal(error.into().to_string())),
            }
        }
    }
}
```

```rust
async fn authenticate_sse_c_parts(
    kubo: &crate::kubo::KuboClient,
    parts: &[(String, i64)],
    key: Arc<crate::crypto::ObjectKey>,
) -> S3Result<()> {
    for (_, expected_size) in parts {
        if *expected_size < 0 {
            return Err(s3s::s3_error!(InvalidPart, "invalid recorded SSE-C part size"));
        }
    }

    for (cid, expected_size) in parts {
        let stream = crate::kubo::cat::stream_cat(kubo, cid, None)
            .await
            .map_err(|error| s3s::s3_error!(InternalError, "cat: {error}"))?;
        let decrypted = crate::crypto::chunker::decrypt_chunk_stream(stream, key.clone());
        tokio::pin!(decrypted);
        let mut observed: i64 = 0;
        while let Some(chunk) = decrypted.next().await {
            match chunk {
                Ok(bytes) => {
                    observed = checked_part_plaintext_size(observed, bytes.len())?;
                    if observed > *expected_size {
                        return Err(s3s::s3_error!(InvalidPart, "decrypted part exceeds recorded size"));
                    }
                }
                Err(crate::error::AppError::Crypto(_)) => {
                    return Err(s3s::s3_error!(InvalidPart, "SSE-C part authentication failed"));
                }
                Err(error) => {
                    return Err(s3s::s3_error!(InternalError, "encrypted part stream: {error}"));
                }
            }
        }
        if observed != *expected_size {
            return Err(s3s::s3_error!(InvalidPart, "decrypted part size does not match metadata"));
        }
    }
    Ok(())
}

fn checked_part_plaintext_size(observed: i64, chunk_len: usize) -> S3Result<i64> {
    let chunk_len = i64::try_from(chunk_len)
        .map_err(|_| s3s::s3_error!(InvalidPart, "decrypted part length exceeds i64"))?;
    observed.checked_add(chunk_len).ok_or_else(|| {
        s3s::s3_error!(InvalidPart, "decrypted part length overflow")
    })
}
```
